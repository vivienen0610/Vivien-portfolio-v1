設計方法論

貫穿 5 隻 Agent 的共通設計原則

這些是我在每一隻 Agent 都堅持的工程準則——也是讓 AI 產出「能放心拿去提案」的關鍵。

① 數據可溯源、反幻覺

每個具體數字與聲明都必須能對應到某次工具實際回傳;查不到就明確標「數據缺失」,嚴禁 LLM 自行推算或用「大約／估計」帶過。

② 確定性運算優先於 LLM

分類、過濾、日期計算等改用 Python／regex 處理（可重現），不交給 LLM 自由發揮,降低不可控的隨機性。

③ 多源工具編排

串接 Redash、Superset BigQuery、Dcard API、Google Drive、Slack 等多個系統,並以「平行查詢」縮短等待。

④ 品質與安全護欄

業配文／成人內容程式化過濾、對外輸出數據模糊化、強制套用品牌視覺規範——確保產出可直接對外。

流程型

A2・品牌 / TA 輿情洞察 Agent

給定品牌或受眾輪廓，挖掘人口輪廓、興趣行為、主動搜尋需求與真實留言心聲
▾

Before

人工讀大量文章與留言整理 TA 洞察，主觀又耗時，業配文混入還會污染結論。

After

多 Query 平行查詢 + 留言自動萃取 + 業配文程式化剔除 → 五維度 TA 洞察報告。

獨有亮點（流程）
  • 🛡️業配文剔除演算法：偵測留言中的分潤連結(regex),只要一則命中就把整篇從名單移除、自動補下一順位,避免養留言的業配污染洞察
  • 📐嚴謹查詢治理：所有查詢走「唯一允許流程」、四種固定回執格式、失敗自動 fallback,讓結果可稽核
  • 🔭跨看板異常捕捉：找品牌在「非預期看板」的討論,挖出隱性需求場景
  • 🧠我設計：業配文剔除邏輯、查詢治理流程與回執規範、跨看板捕捉策略、數據溯源表
  • 🔧我串接：Redash（多個查詢）、Dcard API（文章 / 留言）
  • 🤖AI 生成：TA 需求洞察文字、留言心聲歸納
迭代故事：發現「養留言」的業配文會污染 TA 洞察 → 加入「任一留言含分潤連結即整篇剔除並補位」的機制。
流程型

A3・趨勢預測 × 切角發想 Agent

為品牌指定的未來檔期，預測值得提前佈局的話題，產出切角與可貼上的文案
▾

Before

憑經驗猜趨勢，缺數據依據，難說服客戶為何此時操作這個話題。

After

用數據交叉驗證後的切角建議 + 直接可貼上的 Dcard 文案，提案更有底氣。

獨有亮點（方法論）
  • 🎯雙區間交叉驗證預測法：用「去年同期聲量 × 近期動能」交叉成四象限,決定佈局力度(強推 / 輕量 / 試水 / 不做),把「預測」變成有依據的判斷
  • 🔒反臆測：每個預測結論都強制附參考區間數據,禁止「憑感覺」;目標區間尚未發生時強制標注,不准填推測數字
  • 💡嚴謹 + 發散並行：數據連結型切角(有憑有據) 與 天馬行空創意切角(發散) 同時產出
  • 🧠我設計：雙區間交叉驗證方法論、切角產出框架、話題能量公式展開邏輯
  • 🔧我串接：Redash、Dcard API、Web Search（外部趨勢）
  • 🤖AI 生成：切角創意、可貼上的 Dcard 文案

Before

After

獨有亮點（成果）
  • 🧩自建「話題黃金公式」五層分析框架：把模糊的「為什麼會爆紅」拆成 時機 × 共鳴 × 格式 × 擴散 × 長尾 + 品牌出現層位,讓成功變成可複製的結構
  • 📄成果可直接用：拆解報告的「成功關鍵 + 如何套用到下一個提案」段落,業務拿了就能用
  • 🧠我設計：五層黃金公式框架、案例拆解輸出格式、心理學切角工具箱
  • 🔧我串接：Web Search、Dcard API、內部案例知識庫
  • 🤖AI 生成：案例五層拆解分析、可複製邏輯萃取
成果樣本：此處放一份「案例拆解報告」截圖（去識別化），讓人一眼看到產出長相。
